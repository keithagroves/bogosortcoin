BOGOSORTCOIN

Proof-of-work, secured by a shuffle that has to land back in order.

Mining succeeds when a hash-derived permutation of N numbers comes back sorted — odds of 1/N! per attempt — checked alongside a normal hash-target comparison for fine-grained difficulty. SHA3-256 and SHAKE256 do the actual work; the sort is just how the result gets read.

How a mining attempt works

  1. Hash the candidate block header with a nonce baked in: seed = SHA3-256("BOGOPOW-v1/seed" || header).
  2. Stretch that seed into randomness with SHAKE256("BOGOPOW-v1/permutation" || seed) and run Fisher-Yates against it, using rejection sampling so no value is more likely than any other.
  3. Check two things: the permutation has to equal [0, 1, 2, ..., N-1] exactly, and a second hash (the ticket) has to fall under the current difficulty target.

Why there's no shortcut

A miner only controls the nonce. Change it and the seed changes completely, so there's no way to work backward from "I want the sorted permutation" to a nonce that produces it, short of breaking SHA3-256 preimage resistance. Faster hardware gets you more attempts per second, same as any hash-based proof-of-work. It doesn't get you better odds per attempt.
